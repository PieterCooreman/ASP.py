Quick Start

Up and running in under two minutes. Point it at your existing .asp directory.

1 Install dependencies
# Install optional dependencies as needed
pip install fpdf2 bcrypt pillow pyodbc
2 Start the server
# Syntax: python -m ASPPY.server <host> <port> <webroot>
python -m ASPPY.server 0.0.0.0 8080 www
3 Open your browser
# Your .asp pages are now live!
http://localhost:8080
Requirements
  • Python 3.8+ — any modern Python installation
  • fpdf2 — for PDF generation features
  • bcrypt — for password hashing
  • pillow — for image processing
  • pyodbc — for ODBC database connectivity

Frequently Asked Questions

Everything you need to know about ASPPY.

No. ASPPY runs on Windows, Linux, and macOS. In fact, Linux typically runs Python 10–30% faster than Windows, so you may get better performance by migrating to Linux.

Most legacy ASP applications work as-is. ASPPY implements the full Classic ASP object model with near-complete VBScript built-in coverage. Locale-specific formatting, edge-case type coercion, and COM-level quirks may differ. Run your own regression tests before cutting over critical applications.

ASPPY supports SQLite, MS Access, Excel (read-only), ODBC, and PostgreSQL. SQL is executed as-is by the underlying driver — no dialect translation is performed, so your existing queries should work.

Beyond classic ASP, ASPPY ships JSON encode/decode, ZIP file handling, PDF generation (via fpdf2), image processing (via Pillow), and bcrypt password hashing — all accessible from your VBScript code via the global ASPPY object.
